 Тестове завдання

Побудувати сервіс для реєстрації Заявок на отримання готівки у відділенні банку. 
Сервіс має бути підготовленим для запуску в docker. Архітектура сервісу наступна:
	WebApi що реалізує вхідну точку.(asp.net core 3.1) 
	Задача Api: прийняти запит, залогувати його та доповнити даними про ip клієнта що його надіслав. 
	Отриманий об’єкт переслати на обробку в бек за допомогою RabbitMq.(Обов’язкова обробка помилок та їх логування)

Запит на збереження заявки:
{
    client_id: “<some client id>”,
    departemnt_address: “<address>”,
    amount: 1000.00,
    currency: “UAH(or else)”
}

Запит на отримання статусу заявки №1:
{
    request_id: <id заявки> 
}


Запит на отримання статусу заявки №2:
{
    client_id: “<some client id>”,
    departemnt_address: “<address>”
}


2) Бековий обробник запитів. Вичитує запит з Rabbit, логує та: 
	зберігає його до БД MsSql. 
	У відповідь повертається id замовлення що відповідає id запису до БД.(Обов’язкова наявність в БД поля що відповідає за стан заявки готова до видачі або інше)
	за id запису або client_id + department_address повертає масив із сумою, валютою і статусом заявки.(Заявок може бути декілька. Наприклад з різними валютами)

Роботу з БД реалізувати на stored procedure та через Dapper. Скрипт створення табличок у БД та stored_precedures надати у вигляді txt файлу.
